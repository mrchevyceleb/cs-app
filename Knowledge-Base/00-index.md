# R-Link Customer Service Knowledge Base -- Master Index

## Overview

Welcome to the R-Link Customer Service Knowledge Base. This documentation suite provides developer-level expertise on the R-Link live video collaboration platform, built on the Base44 platform. R-Link supports three session types -- Meeting, Webinar, and Live Stream -- across Basic and Business subscription plans.

This knowledge base is designed for AI customer service agents and human support staff. Every document follows a consistent structure with overview, step-by-step guides, settings, troubleshooting, FAQ, known limitations, plan requirements, and cross-references. Use the question routing table below to quickly locate the correct document for any customer inquiry.

## Emergency Escalation Paths

### Tier 1 -- AI Agent / Frontline Support
- Account setup, plan questions, navigation help
- Common troubleshooting (device issues, browser compatibility)
- Feature explanations and how-to guidance
- Password resets and basic authentication issues
- **Resolution target:** Immediate to 15 minutes

### Tier 2 -- Technical Support
**Escalate when:**
- Customer reports persistent streaming failures after standard troubleshooting
- Integration connections fail repeatedly with correct credentials
- Recording files are corrupted or missing from storage
- Account shows incorrect plan limits after upgrade
- Custom RTMP endpoints are not receiving streams
- WebRTC connectivity failures in specific network environments
- **Resolution target:** 1-4 hours

### Tier 3 -- Engineering / Platform Team
**Escalate when:**
- Base44 platform errors (500-series) affecting multiple users
- Data inconsistency between entities (e.g., recordings exist but no associated event)
- SDK-level authentication failures not resolved by token refresh
- WebRTC infrastructure outages
- RTMP relay service failures
- Billing system discrepancies (charges not matching plan)
- Security incidents or suspected unauthorized access
- **Resolution target:** 4-24 hours

### Escalation Information to Collect
Before escalating, always gather:
1. Customer email and account identifier
2. Current plan (Basic/Business)
3. Browser name and version
4. Operating system
5. Steps to reproduce the issue
6. Error messages (exact text or screenshots)
7. Timestamp of when the issue occurred
8. Network environment (corporate firewall, VPN, etc.)
9. Any recent account changes (plan upgrade, new integrations)
10. Console errors if available (F12 > Console tab)

## Document Conventions

### Formatting Standards
- **Bold text** indicates UI element names (buttons, tabs, menu items)
- `Code formatting` indicates technical values (URL parameters, entity fields, API values, localStorage keys)
- "Quoted text" indicates exact UI labels or error messages
- Tables use pipe-delimited markdown format
- Step-by-step procedures use numbered lists
- Lists of options use bullet points

### Plan Indicators
- **(Basic+)** = Available on Basic plan and above
- **(Business)** = Requires Business plan
- **(Owner)** = Requires account owner permission
- **(Admin)** = Requires admin role
- **(Host)** = Requires host role or above

### Status Indicators
- ACTIVE = Feature is currently available and functioning
- BETA = Feature is available but may have limited functionality
- DEPRECATED = Feature is being phased out
- PLANNED = Feature is on the roadmap but not yet available

### Cross-Reference Format
All cross-references use the format: `See [Document Title](filename.md)` or `See [Document Title](filename.md#section-anchor)`.
